1. 单/双击对比：PC逻辑重叠 vs 手机双击放大

系统级冲突

🖱️ 鼠标端：双击必然包含两次单击，会导致单击和双击代码混在一起执行。

👆 触摸端：手机网页默认会拦截快速的“双击”用于放大页面，导致 JS 中的 dblclick 事件极大概率丢失或失效。

👉 体验：在电脑上双击看日志叠加；在手机模式下双击，看页面是否被系统强行放大了。

2. 拖拽对比：鼠标拖拽失效 vs 独立补充 Touch 事件

事件差异

🖱️ 鼠标端：依赖 mousedown / mousemove / mouseup 实现拖拽。

👆 触摸端：鼠标代码在手机上完全瘫痪（摸上去根本拖不动）。必须另外添加 touchstart / touchmove / touchend，并且用代码阻止页面跟随滑动。

👉 体验：随意拖拽橙色块，留意右侧日志：到底是触发了 Mouse 还是 Touch 事件流？

3. 长按对比：自定义长按 vs 拦截系统原生右键菜单

菜单接管

🖱️ 交互目标：按住超过1秒打开画笔设置菜单。

👆 底层冲突：在手机上长按，系统默认会弹出“复制/粘贴/保存图片”的右键菜单。在 PC 上点右键也是如此。必须手动禁止 contextmenu 事件。

👉 体验：按住画笔超1秒。我们已经强行禁用了系统右键菜单，您可以在日志看到拦截记录。
